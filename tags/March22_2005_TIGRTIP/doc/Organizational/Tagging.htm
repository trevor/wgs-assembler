Assembler Tagging Scheme

This document describes the current Assembler tagging scheme used in support of daily and weekly builds and runs. This scheme currently supports three types of tests (see below). Each test set involves checking out the cds/AS tree sources with a particular tag (detailed below) running the script cds/AS/scripts/assembler on a set of inputs.

Periodic Build

This periodic build checks out the tip of the trunk, and attempts to build it. If successful, the checked out revisions are tagged "AS_PRIV_ASSEMBLER_BUILT".

Tags

Scratch Tags

AS_PRIV_ASSEMBLER_BUILT

Code most recently built as part of a periodic build.

AS_PRIV_ASSEMBLER_UNDER_DAILY_TEST

Code that was successfully built as part of most recent daily test

AS_PRIV_ASSEMBLER_RAN_DAILY_TEST

Code that successfully ran as part of most recent daily test

AS_PRIV_ASSEMBLER_UNDER_WEEKLY_TEST

Code that was successfully built as part of most recent weekly test

AS_PRIV_ASSEMBLER_RAN_WEEKLY_TEST

Code that successfully ran as part of most recent weekly test

AS_PRIV_ASSEMBLER_UNDER_DROS_TEST

Code that was successfully built as part of most recent DROS test

AS_PRIV_ASSEMBLER_RAN_DROS_TEST

Code that successfully ran as part of most recent DROS test

 

Dated Tags

 

 

AS_PRIV_ASSEMBLER_RAN_DROS_TEST_<date>

Code that successfully ran as part of the DROS test on <date>

AS_PRIV_ASSEMBLER_RAN_ WEEKLY_TEST_<date>

.Code that successfully ran as part of the weekly test on <date>

AS_PRIV_ASSEMBLER_RAN_DAILY_TEST_<date>

.Code that successfully ran as part of the daily test on <date>
